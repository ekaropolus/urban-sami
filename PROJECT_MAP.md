# Project Map for GitHub Review

This file connects the final monograph to the executable scripts, report folders, and persistent database artifacts of the `urban-sami` project.

The goal is simple: a reviewer should be able to start from a phase in the monograph, open the script that produced it, inspect the report pack, and, when applicable, verify which persistent tables were written to the experiment database.

## How To Review This Repository

1. Pick a phase in the monograph.
2. Find the matching row below.
3. Open the listed script or scripts.
4. Open the corresponding `report.md` and CSV outputs in `reports/`.
5. If the phase writes persistent results, inspect the listed `derived.*` or `experiments.*` tables in the independent experiment database.

Most scripts can be run as:

```bash
cd "$(git rev-parse --show-toplevel)"
PYTHONPATH=src python3 scripts/<script_name>.py
```

Some ingestion/bootstrap steps additionally require the independent database to exist first:

```bash
bash scripts/bootstrap_experiment_db.sh
```

By default, the lightweight GitHub snapshot does not track `data/`, `reports/`, `logs/`, or `dist/`. Those artifact directories are regenerated locally. See [docs/REPRODUCIBILITY.md](docs/REPRODUCIBILITY.md).
