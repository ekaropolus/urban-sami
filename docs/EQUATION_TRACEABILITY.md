# Equation Traceability

This document maps the main mathematical objects in the final monograph to the scripts, database tables, and report packs that produced them.

It is meant for scientific review. A reviewer should be able to read an equation in the monograph, find the corresponding row here, and then inspect the code path that generated the estimates or materialized the object.

Companion machine-readable file:

- [equation-traceability.csv](equation-traceability.csv)

## Scope

The monograph contains many displayed equations. This traceability file covers all model-defining equations, derived variables, residual objects, information-theoretic objects, decomposition identities, and summary outcomes. Repeated numeric result displays are mapped to the same equation object that generated them.

## How To Use This File During Review

1. Open the monograph equation or subsection.
2. Find the closest equation ID above.
3. Open the listed script.
4. Inspect the listed persistent table or report pack.
5. If the equation is a model family, inspect the model summary CSV or DB table before interpreting the coefficient signs.

## Notes

- A single script can map to several equations because most phases define a data object, fit a model, and then derive residuals or diagnostics.
- Some equations are interpretive statements rather than fitted regressions. Those are still traced to the scripts or reports that motivated the interpretation.
- Generated folders such as `reports/` and `data/` are ignored in the lightweight GitHub snapshot. They are reproducible outputs and are documented through `PROJECT_MAP.md` and `docs/REPRODUCIBILITY.md`.
